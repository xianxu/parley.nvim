# loom: create a subsystem that uses claude to manage durable projects

I realize `claude`, `codex` is pretty good way to manage long running project, with project's internal state kept in md files in a folder, and todos, and time they should be triggered or due encoded in some format. I guess essentially open crawl? but 1/ repo based; 2/ markdown files to make state visible to human inspection and iteration; 3/ use stock coding agent's capability for multi-step reasoning, externalize its mental states in md, seems to fit knowledge worker more. 

In an analogy, parley already showed the start of this pattern:

1/ parley chat, brainstorm tool, it keeps state globally in chat files, but more recent we allow move chat tree into other directories, and the whole point was to move chat to repo's design/ directory as the output of brainstorming. 

2/ issue system, keep things related in issues/, then a hybrid of scripts (created by claude) and claude itself (during coding sessions) to manipulate the state to form a workflow. 

3/ vision, the new roadmapping tool I'm tinkering, storing states (yaml) in vision/, then currently using scripts (generated by claude) to manipulate. 

Now: 

4/ loom, to support more wide variety of tasks, but at its core, leveraging coding agents, to manage some workflows, keeping states in repo. 

Those together, forms the new "operating system" of modern startup.
