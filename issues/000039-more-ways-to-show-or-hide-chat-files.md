# more ways to show or hide chat files

As I use parley more, there are a lot of chat files. I want easy way to find them, or hide them from search to reduce the noise. right now I'm thinking about the following. 

1. we filter based on [tag], this can already be done manually by query [tag] in chat finder. 
2. we can make those even more intuitive, by display a bar of tag at the bottom of chat finder, but above the search box, e.g. [AI] [Tech] ...
3. those tags are extracted from what's under current search criteria.
4. the tags in this bar follow a distinct semantic: 
  1. they are like button, that can be clicked. 
  2. click on their toggle their state
  3. they are on by default at the start
  4. they represent a search, connected with OR. so initially it search for all tags OR-ed together. 
  5. a special tag [] is reserved to represent "no tag", so if you click on it, it will search for files without any tags.
  6. when a [tag] button is unselected, it is effectively an negation, the portion of query generated by tag bar become [AI] OR [Tech]) AND (NOT [Programming]) AND (NOT [Life]). If a file has [Tech], [Programming] as tag, it would be excluded.
  7. tag bar is combined with query using an AND relationship.
  8. untagged file can be done by adding one "hidden" [] search term to each file without tag, and then query [] would match all untagged file


So the search behavior becomes a "proximal prefix search" AND some boolean logic on tags.
