---

### Project: `parley.nvim`

A **Neovim plugin** that provides a streamlined LLM chat interface with syntax highlighting and navigation features. It supports multiple AI providers:
- **OpenAI** (and OpenAI-compatible backends)
- **Anthropic** (Claude)
- **Google AI** (Gemini)
- **Ollama** (local models)

---

### Architecture Breakdown

The project is organized into well-defined domains, each with its own spec in `specs/`:

#### 1. Core Chat System
Handles the full chat lifecycle — format, parsing, memory, branching. Notable: it supports **inline branch links** (`[🌿:text](file)`) for conversation trees, and a **memory/summarization** system.

#### 2. LLM Providers & Agents
A transport-layer abstraction over multiple backends. There's a `CLIProxyAPI` provider — a local proxy for multi-vendor model access. Agent configurations and **editable system prompts** are also part of this layer.

#### 3. Context & References
Supports pulling in context from local files (`@@` syntax), **Google Drive/Docs** (with OAuth), and **web search** tools.

#### 4. Notes & Templates
A note-taking subsystem with year/month/week organization and a template system — suggests this plugin doubles as a personal knowledge tool, not just a chat interface.

#### 5. Issue Management
Repo-local issue tracking with a per-issue file format, scheduler, and decomposition support. This is a fairly unusual feature for a chat plugin.

#### 6. UI/UX
Custom floating-window pickers, outline navigation, lualine statusline integration, key binding help, and syntax highlighting groups.

#### 7. Infrastructure
- **Vault** for secret/API key management
- **Repo Mode** — marker-file detection to activate plugin features per-repo
- **OpenShell Sandbox** — policy-enforced agent sandbox via NVIDIA OpenShell + mutagen file sync (suggests agentic code execution capabilities)
- **AI Workflow** — issue-based development with worktree management

#### 8. Special Modes
- **Interview Mode** — with automatic timestamps
- **Raw Mode** — for debugging raw API requests/responses

#### 9. Export
- Jekyll HTML and Markdown export
- **Tree Export** — exporting branched chat trees as multiple linked files

#### 10. Vision Tracker
YAML-based vision/goal tracking with CSV and DOT graph export — another unusual feature suggesting project management integration.

---

### Implementation Structure (`lua/parley/`)

The Lua source is quite extensive (~40+ modules), covering everything from `chat_parser.lua`, `providers.lua`, `tool_loop.lua`, to `vision.lua` and `issues.lua`. There's also a `tools/builtin/` directory with file system tools (`read_file`, `grep`, `find`, etc.) — confirming **agentic tool-use** capabilities where the LLM can interact with the filesystem.

---

### Overall Impression

This is a **serious, feature-rich plugin** that goes well beyond a simple chat UI. It appears to be a personal **AI-augmented development and knowledge environment** built inside Neovim — combining chat, agents with tool use, note-taking, issue tracking, and project vision management.
